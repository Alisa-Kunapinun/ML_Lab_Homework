Lab 05: Optimization Using Newton's Method¶

In this lab, we'll explore an alternative to gradient descent for nonlinear optimization problems: Newton's method.

Newton's method in one dimension¶

Consider the problem of finding the roots $\textbf{x}$ of a nonlinear function $f: \mathbb{R}^N \rightarrow \mathbb{R}$. A root of $f$ is a point $\mathbf{x}$ that satisfies $f(\mathbf{x}) = 0$.

In one dimension, Newton's method for finding zeroes works as follows:

  1. Pick an initial guess $x_0$

  2. Let $x_{i+1} = x_i + \frac{f(x_i)}{f'(x_i)}$

  3. If not converged, go to #2.

Convergence occurs when $|f(x_i)| < \epsilon_1$ or when $|f(x_{i+1})-f(x_i)| < \epsilon_2$.

Let's see how this works in practice.

Newton's method for optimization¶

Now, consider the problem of minimizing a scalar function $J : \mathbb{R}^n \mapsto \mathbb{R}$. We would like to find $$ \theta^* = \text{argmin}_\theta J(\theta) $$ We already know gradient descent: $$ \theta^{(i+1)} \leftarrow \theta^{(i)} - \alpha \nabla_J(\theta^{(i)}).$$ But Newton's method gives us a potentially faster way to find $\theta^*$ as a zero of the system of equations $$\nabla_J(\theta^*) = \mathbf{0}.$$

In one dimension, to find the zero of $f'(x)$, obviously, we would apply Newton's method to $f'(x)$, obtaining the iteration $$x_{i+1} = x_i - f'(x_i) / f''(x_i).$$ The multivariate extension of Newton's optimization method is $$\mathbf{x}_{i+1} = \mathbf{x}_i - \mathtt{H}^{-1}_f(\mathbf{x}_i) \nabla_f(\mathbf{x}_i),$$ where $\mathtt{H}_f(\mathbf{x})$ is the Hessian of $f$ evaluated at $\textbf{x}$: $$\mathtt{H}_f(\mathbf{x}) = \begin{bmatrix} \frac{\partial^2 f}{\partial x_1^2} & \frac{\partial^2 f}{\partial x_1 x_2} & \cdots & \frac{\partial^2 f}{\partial x_1 x_n} \\ \frac{\partial^2 f}{\partial x_2 x_1} & \frac{\partial^2 f}{\partial x_2^2} & \cdots & \frac{\partial^2 f}{\partial x_2 x_n} \\ \vdots & \vdots & \ddots & \vdots \\ \frac{\partial^2 f}{\partial x_n x_1} & \frac{\partial^2 f}{\partial x_n x_2} & \cdots & \frac{\partial^2 f}{\partial x_n^2} \end{bmatrix}$$

This means, for the minimization of $J(\theta)$, we would obtain the update rule $$\theta^{(i+1)} \leftarrow \theta^{(i)} - \mathtt{H}^{-1}_J(\theta^{(i)}) \nabla_J(\theta^{(i)}).$$

class Logistic_NM: #logistic regression for newton's method
      
    def __init__(self):
        pass
    
    def sigmoid(self,z):
        s = 1/(1+np.exp(-z))
        # YOUR CODE HERE
        #raise NotImplementedError()
        return s

    def h(self, X, theta):
        hf = self.sigmoid(np.dot(X,theta))
        # YOUR CODE HERE
        #raise NotImplementedError()
        return hf

    def gradient(self, X, y, y_pred):
        grad = -X.T@(y-y_pred)/y.shape[0]
        # YOUR CODE HERE
        #raise NotImplementedError()
        return grad
        
    def hessian(self, X, y, theta):
        y_pred = self.h(X, theta)
        #print("y_pred : ", y_pred)
        hess_mat = y_pred.T@(1-y_pred)
        #print("hess_mart: ", hess_mat)
        hess_mat = hess_mat[0][0]*(X.T@X)/y_pred.shape[0]
        #print("hess_mart: ", hess_mat)
        # YOUR CODE HERE
        #raise NotImplementedError()
        return hess_mat
        
    def costFunc(self, theta, X, y):   
        cost = np.array(np.sum(-y*np.log(self.h(X,theta))-(1-y)*np.log(1-self.h(X,theta)))/X.shape[0])
        grad = self.gradient(X, y, self.h(X,theta))
        # YOUR CODE HERE
        #raise NotImplementedError()
        return cost, grad
    
    def newtonsMethod(self, X, y, theta, num_iters):
        m = len(y)
        J_history = []
        theta_history = []
        for i in range(num_iters):
            # YOUR CODE HERE
            cost, grad = self.costFunc(theta, X, y)
            theta = theta - np.linalg.inv(self.hessian(X,y,theta))@grad
            #raise NotImplementedError()
            
            J_history.append(cost)
            theta_history.append(theta)
        J_min_index = np.argmin(J_history)
        print("Minimum at iteration:", J_min_index)
        return theta_history[J_min_index] , J_history

    def predict(self,X, theta):
        labels=[]
        y_predict = self.h(X,theta).argmax(axis=1).astype(int)
        #print("y_predict: ", y_predict)
        for i in range(0, len(y_predict)):
            if y_predict[i] == 1:
                labels.append(2)
            elif y_predict[i] == 0:
                labels.append(1)
        
        labels=np.asarray(labels)
        return labels
     
    def getAccuracy(self,X,y,theta):
        y_pred = np.round( self.h(X,theta) )
        #print("y_pred: ", y_pred)
        print("y_pred shape: ", y_pred.shape)
        print("y: ", y.shape)
        #print("y: ", y)
        percent_correct = np.sum(y==y_pred)
        print(percent_correct)
        size = y_pred.shape[0]
        percent_correct = (percent_correct / size)*100
        return percent_correct

Exercise 2.3 (5 points)¶

Compare the number of iterations required for gradient descent vs. Newton's method. Do you observe other issues with Newton's method such as a singular or nearly singular Hessian matrix?

Student's answer Score: 5.0 / 5.0 (Top)

The number of iterations required by Logistic Regression method was 50,000. The same results were achieved by Newton's Method with only 1000 iterations (actually earlier between 500-600 iterations where the loss curve became flat).

The Hessian matrix calculated above has a non-zero determinant therefore it is not singular. However, if the Hessian matrix was singular then it would not have an inverse and we could not use Newton's Method to solve the Logistic Regression Problem. Also we need the matrix to be full rank, which means we must have a lot more samples than features. Also each sample must be linearly independant of each other.

        return

    labels = [x for x in data[selected_column].unique() if x == x]
    print("labels: ", labels)
    #print("Column Values: ", column_values.values)
    
    for label in labels:

        num_label = round(ratio_label * num_nullValues)        
        print(f"For label: {label} --> ratio_label: {ratio_label}, num_label: {num_label}")

    print('\n')
    assert data[selected_column].isnull().sum() == 0

When we have data which is linearly inseparable, the classifiers have difficulty distinguishing between the classes. The boundary line in this case is not the most optimum and gives poor accuracy results.

We can classify linearly unseparable data using other models such as naive bayes', SVM using kernels, or the various tree methods. However, the same logistic regression and Newton-Rhaphson Methods may also be used if we convert the data to another coordinate space, where the two classes become linearly separable.

In this Lab, to separate linearly inseparable data (circular data), we convert it to the Polar coordinate space. In the Polar coordinate space, the data becomes linearly separable as the phi (angle) feature converts angularity to the linear domain.

We see that after converting to the Polar coordinate space we are able to classify data with a high accuracy. Same is reflected in the plot where boundary line is drawn between the two classes.

For the data from Lab03 which we solved using Logistic Regression, we import the Loan Prediction Data and fill all null values using mean values. We then encode our categorical data to numerics. Data normalization is performed to enable faster convergence.

We see that the Newton-Rhaphson method gives us the same accuracy as the Logistic Regression algorithm we used in Lab03 (accuracy=0.808).
